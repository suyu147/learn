## 📖 核心概念

### 时间复杂度

大 O 表示法描述算法运行时间随输入规模增长的上界：

| 复杂度 | 名称 | n=100 时的操作数 | 典型算法 |
|:------:|------|:---------------:|---------|
| O(1) | 常数 | 1 | 哈希表查找 |
| O(log n) | 对数 | ~7 | 二分查找 |
| O(n) | 线性 | 100 | 线性扫描 |
| O(n log n) | 线性对数 | ~664 | 归并排序 |
| O(n²) | 平方 | 10,000 | 冒泡排序 |
| O(2ⁿ) | 指数 | ~10³⁰ | 递归求子集 |
| O(n!) | 阶乘 | ~10¹⁵⁸ | 旅行商问题 |

### 排序算法对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 | 特点 |
|------|:------:|:------:|:----:|:----:|------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | 稳定 | 简单但慢 |
| 选择排序 | O(n²) | O(n²) | O(1) | 不稳定 | 交换次数少 |
| 插入排序 | O(n²) | O(n²) | O(1) | 稳定 | 近乎有序时 O(n) |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 稳定 | 分治经典 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 | 实际最快 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 不稳定 | 原地排序 |

### 二分查找

在有序数组中查找目标值，每次将搜索范围减半：

$$ T(n) = T\left(\frac{n}{2}\right) + O(1) \implies O(\log n) $$

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

### 动态规划

动态规划的两个核心要素：
1. **最优子结构**：问题的最优解包含子问题的最优解
2. **重叠子问题**：子问题被重复计算

**经典问题 — 背包问题 (0-1 Knapsack)**：

$$ dp[i][w] = \max(dp[i-1][w], dp[i-1][w-w_i] + v_i) $$

### 贪心算法

贪心算法每一步都做当前最优选择，适用于具有**贪心选择性质**的问题：

| 问题 | 贪心策略 | 最优？ |
|------|---------|:----:|
| 活动选择 | 最早结束 | ✅ |
| 霍夫曼编码 | 最小频率合并 | ✅ |
| 找零钱 | 最大面额 | 取决于币种 |
| 背包问题 | 最高价值/重量 | ❌ (需 DP) |

### 算法设计范式

```
算法设计范式
├── 分治法 (Divide & Conquer)
│   └── 分解→解决→合并 (归并排序、快速排序)
├── 动态规划 (Dynamic Programming)
│   └── 自顶向下(记忆化) / 自底向上(填表)
├── 贪心算法 (Greedy)
│   └── 局部最优 → 全局最优
├── 回溯法 (Backtracking)
│   └── 试错→剪枝 (八皇后、数独)
└── 分支限界 (Branch & Bound)
    └── BFS + 剪枝 (旅行商问题)
```

### 常见算法模板

```
双指针:    数组去重、两数之和、滑动窗口
快慢指针:  环形链表检测、链表中点
```

## 💻 代码实现

```python
# ===== 快速排序 =====
def quicksort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quicksort(left) + middle + quicksort(right)

# ===== 动态规划: 最长递增子序列 (LIS) =====
def length_of_lis(nums):
    n = len(nums)
    dp = [1] * n
    
    for i in range(n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)

# ===== 动态规划: 0-1 背包问题 =====
def knapsack(weights, values, capacity):
    n = len(weights)
    # dp[i][w]: 前 i 个物品, 容量 w 的最大价值
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    
    for i in range(1, n + 1):
        w, v = weights[i-1], values[i-1]
        for j in range(capacity + 1):
            if j < w:
                dp[i][j] = dp[i-1][j]
            else:
                dp[i][j] = max(dp[i-1][j], dp[i-1][j-w] + v)
    
    return dp[n][capacity]

# ===== BFS 最短路径 =====
from collections import deque

def bfs_shortest_path(graph, start, target):
    if start == target:
        return 0
    
    visited = {start}
    queue = deque([(start, 0)])  # (node, distance)
    
    while queue:
        node, dist = queue.popleft()
        
        for neighbor in graph.get(node, []):
            if neighbor == target:
                return dist + 1
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, dist + 1))
    
    return -1  # 不可达

# ===== 测试 =====
print("快速排序:", quicksort([3, 6, 8, 10, 1, 2, 1]))
print("LIS 长度:", length_of_lis([10, 9, 2, 5, 3, 7, 101, 18]))  # 4

weights = [2, 3, 4, 5]
values = [3, 4, 5, 6]
print("背包最大价值:", knapsack(weights, values, 8))  # 10

graph = {
    0: [1, 2], 1: [0, 3], 2: [0, 3],
    3: [1, 2, 4], 4: [3]
}
print("最短路径 0→4:", bfs_shortest_path(graph, 0, 4))  # 3
```

## 🎯 关键要点
- 时间复杂度关注增长趋势，不是精确时间
- 快速排序在实际中通常最快，但最坏 O(n²)
- 动态规划的核心是找到状态转移方程
- 贪心正确性需要证明，不是所有问题都能贪心
- 80% 的算法面试题可以用 BFS/DFS/DP/双指针/滑动窗口解决

## ❓ 常见问题

Q: 什么时候用动态规划而不是贪心？
A: 当局部最优不能推出全局最优时。如果问题具有最优子结构但无贪心选择性质，用 DP。

Q: 排序算法的稳定性重要吗？
A: 多关键字排序时需要稳定性（先按 A 排序再按 B 排序，B 相同时 A 的顺序不变）。

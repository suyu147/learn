# 自然语言处理 (NLP) 基础

## 📋 学习目标
- 理解文本预处理的基本流程
- 掌握词向量 (Word Embedding) 的原理
- 理解 RNN/LSTM/GRU 处理序列数据的机制
- 了解 Seq2Seq 和注意力机制
- 能够使用 PyTorch 实现情感分类模型

## 📖 核心概念

### 文本预处理管道

```
原始文本 → 分词 → 清洗 → 标准化 → 向量化 → 模型输入
```

| 步骤 | 方法 | 示例 |
|------|------|------|
| 分词 | jieba (中文), nltk (英文) | "我爱NLP" → ["我","爱","NLP"] |
| 清洗 | 去停用词、标点、HTML标签 | 移除 "的","了","是" |
| 标准化 | 小写化、词干提取、词形还原 | "running" → "run" |
| 向量化 | Word2Vec, GloVe, BERT | "猫" → [0.2, -0.5, 0.8, ...] |

### 词向量 (Word Embedding)

词向量将单词映射到低维稠密向量空间，语义相近的词在空间中距离更近：

$$ \text{similarity}(w_1, w_2) = \cos(\mathbf{v}_{w_1}, \mathbf{v}_{w_2}) = \frac{\mathbf{v}_{w_1} \cdot \mathbf{v}_{w_2}}{\|\mathbf{v}_{w_1}\| \|\mathbf{v}_{w_2}\|} $$

**经典关系**：$\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}} + \mathbf{v}_{\text{woman}} \approx \mathbf{v}_{\text{queen}}$

### Word2Vec

Word2Vec 有两种训练模式：

**CBOW (Continuous Bag of Words)**：用上下文预测目标词
**Skip-gram**：用目标词预测上下文

```
CBOW:  [w_{t-2}, w_{t-1}, w_{t+1}, w_{t+2}] → w_t
Skip:  w_t → [w_{t-2}, w_{t-1}, w_{t+1}, w_{t+2}]
```

### RNN (循环神经网络)

RNN 用于处理序列数据，每个时间步共享相同的权重：

$$ h_t = \tanh(W_{hh} h_{t-1} + W_{xh} x_t + b_h) $$
$$ y_t = W_{hy} h_t + b_y $$

**RNN 的局限**：梯度消失导致难以捕捉长距离依赖。

### LSTM (长短期记忆网络)

LSTM 通过门控机制解决长距离依赖问题：

**遗忘门**：$f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f)$
**输入门**：$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i)$
**候选记忆**：$\tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C)$
**更新记忆**：$C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t$
**输出门**：$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o)$
**隐藏状态**：$h_t = o_t \odot \tanh(C_t)$

### GRU (门控循环单元)

GRU 是 LSTM 的简化版本，合并了遗忘门和输入门：

**重置门**：$r_t = \sigma(W_r \cdot [h_{t-1}, x_t])$
**更新门**：$z_t = \sigma(W_z \cdot [h_{t-1}, x_t])$
**候选状态**：$\tilde{h}_t = \tanh(W \cdot [r_t \odot h_{t-1}, x_t])$
**最终状态**：$h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t$

### Seq2Seq 架构

Seq2Seq 由编码器和解码器组成：

```
输入序列 → [编码器 RNN] → 上下文向量 c → [解码器 RNN] → 输出序列
```

注意力机制的引入让解码器每一步可以"关注"编码器的不同位置。

## 💻 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torch.utils.data import DataLoader, Dataset

# 1. 情感分类数据集
texts = [
    "这部电影非常好看我很喜欢",
    "太烂了浪费我的时间",
    "剧情精彩演员演技在线",
    "无聊透顶看不下去",
    "特效震撼值得一看",
]
labels = [1, 0, 1, 0, 1]  # 1=正面, 0=负面

# 2. 构建词表
word2idx = {'<PAD>': 0, '<UNK>': 1}
for text in texts:
    for char in text:
        if char not in word2idx:
            word2idx[char] = len(word2idx)

# 3. LSTM 情感分类模型
class SentimentLSTM(nn.Module):
    def __init__(self, vocab_size, embed_dim=128, hidden_dim=256,
                 output_dim=2, n_layers=2, dropout=0.5):
        super(SentimentLSTM, self).__init__()
        
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(embed_dim, hidden_dim, n_layers,
                           batch_first=True, dropout=dropout,
                           bidirectional=True)
        self.fc = nn.Linear(hidden_dim * 2, output_dim)  # *2 for bidirectional
        self.dropout = nn.Dropout(dropout)
    
    def forward(self, text):
        # text: [batch_size, seq_len]
        embedded = self.dropout(self.embedding(text))  # [B, L, E]
        
        lstm_out, (hidden, cell) = self.lstm(embedded)  # [B, L, 2H]
        
        # 拼接最后的前向和后向隐藏状态
        hidden = torch.cat((hidden[-2,:,:], hidden[-1,:,:]), dim=1)
        hidden = self.dropout(hidden)
        
        return self.fc(hidden)

# 4. 准备数据
class TextDataset(Dataset):
    def __init__(self, texts, labels, word2idx, max_len=50):
        self.texts = texts
        self.labels = labels
        self.word2idx = word2idx
        self.max_len = max_len
    
    def __len__(self):
        return len(self.texts)
    
    def __getitem__(self, idx):
        text = self.texts[idx][:self.max_len]
        encoded = [self.word2idx.get(c, 1) for c in text]
        # Padding
        encoded += [0] * (self.max_len - len(encoded))
        return torch.tensor(encoded), torch.tensor(self.labels[idx])

dataset = TextDataset(texts, labels, word2idx)
dataloader = DataLoader(dataset, batch_size=2, shuffle=True)

# 5. 训练 (演示用)
model = SentimentLSTM(len(word2idx), embed_dim=64, hidden_dim=128)
optimizer = optim.Adam(model.parameters(), lr=0.001)
criterion = nn.CrossEntropyLoss()

print(f"词表大小: {len(word2idx)}")
print(f"模型参数量: {sum(p.numel() for p in model.parameters()):,}")

for epoch in range(10):
    for texts_batch, labels_batch in dataloader:
        optimizer.zero_grad()
        output = model(texts_batch)
        loss = criterion(output, labels_batch)
        loss.backward()
        optimizer.step()
    
    if (epoch + 1) % 5 == 0:
        print(f"Epoch {epoch+1}, Loss: {loss.item():.4f}")
```

## 🎯 关键要点
- 文本预处理（分词→清洗→向量化）是 NLP 的第一步
- 词向量捕获了语义相似性（king - man + woman ≈ queen）
- LSTM 通过门控机制缓解了 RNN 的梯度消失问题
- GRU 是 LSTM 的简化版，效果接近但参数更少
- 注意力机制让模型可以动态关注输入的任意位置

## ❓ 常见问题

Q: LSTM 和 GRU 如何选择？
A: 数据少时 GRU 更不容易过拟合；数据多时 LSTM 稍有优势。实际使用时差异不大。

Q: 为什么 Transformer 取代了 LSTM？
A: Transformer 可以并行计算（不需要逐步处理），且自注意力机制可以捕捉任意距离的依赖。

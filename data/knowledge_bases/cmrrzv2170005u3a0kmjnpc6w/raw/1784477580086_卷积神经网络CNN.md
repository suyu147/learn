# 卷积神经网络 (CNN)

## 📋 学习目标
- 理解卷积操作的基本原理
- 掌握 CNN 的核心组件（卷积层、池化层、全连接层）
- 了解经典 CNN 架构（LeNet, AlexNet, VGG, ResNet）
- 能够使用 PyTorch 构建 CNN 模型

## 📖 核心概念

### 为什么需要 CNN

传统全连接网络处理图像的问题：
- **参数爆炸**：224×224×3 的图，隐藏层 1000 神经元 → 1.5 亿参数
- **忽略空间结构**：全连接将图像展平为一维，丢失空间信息
- **平移不变性**：物体在图中移动，全连接网络无法识别

CNN 通过**局部连接**和**权重共享**解决这些问题。

### 卷积层 (Convolution Layer)

卷积操作的核心思想：使用一个小型滤波器（核）在输入上滑动，计算点积：

$$ (I * K)(i,j) = \sum_{m=0}^{M-1} \sum_{n=0}^{N-1} I(i+m, j+n) \cdot K(m,n) $$

**卷积层参数**：
| 参数 | 含义 | 常用值 |
|------|------|--------|
| kernel_size | 卷积核大小 | 3×3, 5×5 |
| stride | 滑动步长 | 1, 2 |
| padding | 边缘填充 | same, valid |
| filters | 输出通道数 | 32, 64, 128, ... |

**输出尺寸计算**：
$$ H_{out} = \left\lfloor \frac{H_{in} + 2P - K}{S} \right\rfloor + 1 $$

### 池化层 (Pooling Layer)

池化层用于**降采样**，减少特征图尺寸和参数量：

**最大池化 (Max Pooling)**：取区域内的最大值
**平均池化 (Average Pooling)**：取区域内的平均值

```
输入 4×4, pool_size=2, stride=2:
┌───────────┐     ┌──────┐
│1  3  2  4 │     │ 4  5 │   Max Pooling
│5  4  6  2 │  →  │ 8  9 │
│7  8  3  1 │     └──────┘
│2  9  4  5 │
└───────────┘
```

### 感受野 (Receptive Field)

感受野是输出特征图上每个像素对应输入图像的区域大小：

$$ RF_k = RF_{k-1} + (kernel_k - 1) \times \prod_{i=1}^{k-1} stride_i $$

### 批归一化 (Batch Normalization)

$$ \hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} $$
$$ y = \gamma \hat{x} + \beta $$

批归一化的好处：
- 加速训练收敛
- 允许使用更大的学习率
- 减少对初始化的敏感性
- 有一定的正则化效果

### 经典 CNN 架构演进

| 网络 | 年份 | 创新点 | 层数 | Top-5 错误率 |
|------|------|--------|:----:|:------------:|
| LeNet-5 | 1998 | 卷积+池化+全连接 | 7 | - |
| AlexNet | 2012 | ReLU + Dropout + GPU | 8 | 15.3% |
| VGG-16 | 2014 | 堆叠小卷积核(3×3) | 16 | 7.3% |
| GoogLeNet | 2014 | Inception 模块 | 22 | 6.7% |
| ResNet-50 | 2015 | 残差连接 | 50 | 3.6% |
| DenseNet | 2017 | 密集连接 | 121 | - |
| EfficientNet | 2019 | 复合缩放 | B0-B7 | 1.0% |

### 残差连接 (ResNet)

ResNet 的核心创新 — 跳跃连接：

$$ \mathbf{y} = \mathcal{F}(\mathbf{x}, \{W_i\}) + \mathbf{x} $$

当 $\mathcal{F}$ 与 $\mathbf{x}$ 维度不匹配时，使用 1×1 卷积对齐。

## 💻 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# 1. 数据预处理与加载
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize((0.1307,), (0.3081,))
])

train_dataset = datasets.MNIST('./data', train=True, download=True, transform=transform)
test_dataset = datasets.MNIST('./data', train=False, transform=transform)

train_loader = DataLoader(train_dataset, batch_size=64, shuffle=True)
test_loader = DataLoader(test_dataset, batch_size=1000)

# 2. 定义 CNN 模型
class CNN(nn.Module):
    def __init__(self):
        super(CNN, self).__init__()
        # 卷积层
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.bn1 = nn.BatchNorm2d(32)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        self.bn2 = nn.BatchNorm2d(64)
        
        # 池化层
        self.pool = nn.MaxPool2d(2, 2)
        
        # Dropout
        self.dropout = nn.Dropout(0.25)
        
        # 全连接层
        self.fc1 = nn.Linear(64 * 7 * 7, 128)
        self.fc2 = nn.Linear(128, 10)
    
    def forward(self, x):
        # Block 1: Conv → BN → ReLU → Pool
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        # Block 2: Conv → BN → ReLU → Pool
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        
        # Flatten
        x = torch.flatten(x, 1)
        x = self.dropout(x)
        
        # FC layers
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)
        
        return F.log_softmax(x, dim=1)

# 3. 训练
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
model = CNN().to(device)
optimizer = optim.Adam(model.parameters(), lr=0.001)
criterion = nn.CrossEntropyLoss()

def train(model, device, train_loader, optimizer, epoch):
    model.train()
    for batch_idx, (data, target) in enumerate(train_loader):
        data, target = data.to(device), target.to(device)
        optimizer.zero_grad()
        output = model(data)
        loss = criterion(output, target)
        loss.backward()
        optimizer.step()
        
        if batch_idx % 100 == 0:
            print(f'Epoch {epoch}: [{batch_idx*len(data)}/{len(train_loader.dataset)}] '
                  f'Loss: {loss.item():.4f}')

def test(model, device, test_loader):
    model.eval()
    correct = 0
    with torch.no_grad():
        for data, target in test_loader:
            data, target = data.to(device), target.to(device)
            output = model(data)
            pred = output.argmax(dim=1)
            correct += pred.eq(target).sum().item()
    
    accuracy = correct / len(test_loader.dataset)
    print(f'\n测试准确率: {accuracy:.2%}\n')
    return accuracy

# 训练 5 个 epoch
for epoch in range(1, 6):
    train(model, device, train_loader, optimizer, epoch)
    test(model, device, test_loader)
```

## 🎯 关键要点
- CNN 通过局部连接和权重共享大幅减少参数
- 卷积提取局部特征，池化进行降采样
- ResNet 的残差连接解决了深层网络退化问题
- BatchNorm 加速训练并起到正则化作用
- 现代 CNN 架构趋向于更小卷积核 + 更深网络

## ❓ 常见问题

Q: 为什么用 3×3 卷积而不是更大的核？
A: 两个 3×3 卷积的感受野等于一个 5×5 卷积，但参数更少，非线性更多。

Q: 残差连接为什么有效？
A: 跳跃连接让梯度可以直接流过，缓解了深层网络的梯度消失问题。

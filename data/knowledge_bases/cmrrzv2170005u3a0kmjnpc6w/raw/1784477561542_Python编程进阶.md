# Python 编程进阶

## 📋 学习目标
- 掌握 Python 高阶函数和函数式编程
- 理解装饰器的原理和应用
- 掌握生成器和迭代器
- 了解并发编程（多线程、多进程、异步）
- 掌握常用设计模式

## 📖 核心概念

### 函数式编程特性

Python 支持函数式编程范式，函数是一等公民：

| 特性 | 说明 | 示例 |
|------|------|------|
| `map` | 对序列每个元素应用函数 | `map(str, [1,2,3])` |
| `filter` | 按条件过滤序列 | `filter(lambda x: x>0, [-1,0,1])` |
| `reduce` | 累积归约 | `reduce(lambda a,b: a+b, [1,2,3])` |
| 列表推导式 | 简洁的列表生成 | `[x² for x in range(10)]` |
| Lambda | 匿名函数 | `lambda x: x * 2` |

### 装饰器

装饰器是 Python 最强大的语法特性之一，本质是**接受函数并返回新函数的高阶函数**：

```python
@decorator
def func():
    pass

# 等价于: func = decorator(func)
```

**装饰器的核心应用**：
- 日志记录
- 性能计时
- 权限校验
- 缓存 (memoization)
- 重试机制

### 生成器与迭代器

**迭代器**：实现 `__iter__` 和 `__next__` 的对象。
**生成器**：使用 `yield` 的函数，自动创建迭代器。

生成器的优势：
- **惰性计算**：按需生成值，节省内存
- **无限序列**：可以表达无限长的数据流
- **管道组合**：生成器可以串联成数据处理管道

### 并发编程模型对比

| 模型 | 适用场景 | GIL 影响 | 通信方式 | 开销 |
|------|---------|:------:|---------|:----:|
| 多线程 | IO 密集 | 受限 | 共享内存+锁 | 低 |
| 多进程 | CPU 密集 | 不受限 | 管道/队列 | 高 |
| asyncio | 高并发 IO | 不受限 | 协程调度 | 最低 |

**GIL (Global Interpreter Lock)**：CPython 的全局解释器锁，同一时刻只有一个线程执行 Python 字节码。

### 上下文管理器

使用 `with` 语句自动管理资源：

```python
with open('file.txt') as f:
    content = f.read()
# 自动 f.close()，即使发生异常
```

自定义上下文管理器有两种方式：
1. 实现 `__enter__` 和 `__exit__` 方法
2. 使用 `@contextmanager` 装饰器

### 常用设计模式

| 模式 | 核心思想 | Python 示例 |
|------|---------|-----------|
| 单例 | 类只有一个实例 | `__new__` 控制 |
| 工厂 | 解耦对象创建 | `def create_model(type)` |
| 观察者 | 发布-订阅 | `signal/slot` |
| 策略 | 运行时选择算法 | `sorted(key=...)` |
| 装饰器 | 动态添加功能 | `@` 语法糖 |

## 💻 代码实现

```python
import time
import functools
from contextlib import contextmanager
from typing import Any, Generator
import asyncio

# ===== 1. 装饰器: 计时 + 重试 =====
def retry_on_failure(max_retries=3, delay=1):
    """带重试机制的装饰器"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(max_retries):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_retries - 1:
                        raise
                    print(f"重试 {attempt+1}/{max_retries}: {e}")
                    time.sleep(delay)
        return wrapper
    return decorator

@retry_on_failure(max_retries=3)
def unstable_network_call():
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络不稳定")
    return "成功!"

# ===== 2. 生成器: 斐波那契数列 (无限) =====
def fibonacci() -> Generator[int, None, None]:
    """生成无限斐波那契数列"""
    a, b = 0, 1
    while True:
        yield a
        a, b = b, a + b

# 使用: list(itertools.islice(fibonacci(), 10))

# ===== 3. 生成器管道 =====
def read_lines(filename: str):
    """逐行读取"""
    for line in open(filename, encoding='utf-8'):
        yield line.strip()

def filter_empty(lines):
    """过滤空行"""
    for line in lines:
        if line.strip():
            yield line

def parse_csv(lines):
    """解析 CSV"""
    for line in lines:
        yield line.split(',')

# 管道组合
# data = parse_csv(filter_empty(read_lines('data.csv')))

# ===== 4. 上下文管理器 =====
class Timer:
    """计时器上下文管理器"""
    def __enter__(self):
        self.start = time.perf_counter()
        return self
    
    def __exit__(self, *args):
        self.elapsed = time.perf_counter() - self.start
        print(f"耗时: {self.elapsed:.4f}s")

@contextmanager
def temp_env(**kwargs):
    """临时修改环境变量"""
    import os
    old = {}
    for key, value in kwargs.items():
        old[key] = os.environ.get(key)
        os.environ[key] = value
    
    yield
    
    for key, value in old.items():
        if value is None:
            del os.environ[key]
        else:
            os.environ[key] = value

# ===== 5. 单例模式 =====
class Singleton:
    _instance = None
    
    def __new__(cls, *args, **kwargs):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
        return cls._instance

# ===== 6. 异步编程 =====
async def fetch_data(url: str) -> str:
    """模拟异步数据获取"""
    print(f"开始获取: {url}")
    await asyncio.sleep(1)  # 模拟网络 IO
    print(f"获取完成: {url}")
    return f"data from {url}"

async def main():
    # 并发获取多个 URL
    urls = [f"https://api.example.com/data/{i}" for i in range(5)]
    
    # 方案A: gather 并发执行
    results = await asyncio.gather(*[fetch_data(u) for u in urls])
    print(f"获取了 {len(results)} 个结果")
    
    # 方案B: 任务列表
    tasks = [asyncio.create_task(fetch_data(u)) for u in urls]
    done, pending = await asyncio.wait(tasks, timeout=3)
    print(f"完成: {len(done)}, 超时: {len(pending)}")

# ===== 7. 使用 functools =====
@functools.lru_cache(maxsize=128)
def fibonacci_cached(n: int) -> int:
    """带缓存的斐波那契 (记忆化)"""
    if n < 2:
        return n
    return fibonacci_cached(n-1) + fibonacci_cached(n-2)

# ===== 测试 =====
if __name__ == "__main__":
    # 装饰器测试
    try:
        result = unstable_network_call()
        print("网络调用:", result)
    except ConnectionError:
        print("网络调用失败 (已达最大重试)")
    
    # 生成器测试
    fib = fibonacci()
    first_10 = [next(fib) for _ in range(10)]
    print("斐波那契前10项:", first_10)
    
    # 计时器测试
    with Timer():
        _ = sum(range(1000000))
    
    # 缓存测试
    print("fib(30):", fibonacci_cached(30))
    print("缓存信息:", fibonacci_cached.cache_info())
    
    # asyncio 测试
    # asyncio.run(main())
```

## 🎯 关键要点
- 装饰器是 AOP（面向切面编程）在 Python 中的优雅实现
- 生成器在处理大数据流时内存效率极高
- GIL 使 Python 多线程不适合 CPU 密集任务，但 IO 密集仍有效
- `asyncio` 是处理高并发网络 IO 的最佳方案
- `functools.lru_cache` 是无脑添加缓存的利器

## ❓ 常见问题

Q: 什么时候用 `asyncio` 而不是多线程？
A: 网络 IO 密集场景用 `asyncio`（协程开销 < 线程开销），CPU 密集场景用多进程。

Q: 装饰器和 AOP 的关系？
A: 装饰器是 Python 实现 AOP 的方式，可以在不修改原函数的情况下增加横切关注点（日志、计时、权限）。

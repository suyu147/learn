## 修复：学习资源生成失败（apiFetch 信封 unwrap bug）

**现象**：/resources 页面点击任何资源卡片生成，始终显示"生成失败"，但后端 API 实际成功返回。

**根因**：`app/resources/page.tsx` 的 `handleSubmit` / `handleRegenerate` 误用了 `apiFetch` 的返回值。
`apiFetch`（lib/api-client.ts L62-93）会自动对 `{ success, data }` 信封做 unwrap — 成功时返回 `json.data`，失败时抛 `ApiRequestError`。
但页面代码以为返回的是完整 `{ success, data, error }`，写了 `if (!res.success || !res.data)`：
- `res` 实际是 unwrap 后的 `{ type, title, content, metadata }`
- `res.success` 永远 undefined → `!res.success` 恒为 true → 每次都抛 "生成失败"

**修复**：移除 `res.success`/`res.data` 检查，直接用 `res.type`/`res.title`/`res.content`/`res.metadata`。错误由 apiFetch 内部抛出、catch 块捕获。

**验证**：tsc --noEmit -p tsconfig.json 无报错；curl 带 token 测试 code/document/video 三种类型全部成功。

**教训**：使用 `apiFetch` 时，返回值已被自动 unwrap，不要再检查 `success` 字段。这是项目统一的 API 封装约定。

## 修复：quiz 资源 "Failed to parse JSON from response" 控制台错误

**现象**：生成 quiz（习题）类型资源后，QuizPlayer 组件渲染时控制台报 `[Generation] "Failed to parse JSON from response"`。

**根因**：`generateDefaultResource`（lib/learning-graph/helpers/resource-generators.ts）为所有资源类型统一使用 `runTextGeneration` 的默认 `maxOutputTokens=4096`。quiz 的 prompt 要求 5 道题（含 explanation/hints），输出量经常超过 4096 tokens，导致 LLM 输出在 JSON 字符串中间被截断。截断的 JSON 无法被 `parseJsonResponse` 的 4 种策略（代码块提取/括号匹配/整体尝试/jsonrepair）修复，最终打印 log.error。

**修复**：在 `generateDefaultResource` 中为 quiz 和 ppt 类型（均要求 JSON 输出、内容较多）设置 `maxOutputTokens=8192`，其余类型保持 4096。

**验证**：dev server（端口 3001）测试 quiz 生成 — contentLen=3312，Direct JSON.parse 直接成功，无需 jsonrepair。
